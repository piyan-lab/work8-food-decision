# 🧪 手動測試清單 (Test Plan) - Food Decider (v4)

## 2. 手動測試清單

| 測試案例 ID | 測試場景 | 前置條件 | 操作步驟 | 預期結果 |
|---|---|---|---|---|
| **TC-001** | 手動地址搜尋測試 | 已登入 | 1. 手動輸入「台北市中正區忠孝西路一段1號」<br>2. 點擊搜尋 | 成功找到資料庫中內建的測試餐廳，並顯示直線距離 |
| **TC-002** | 模糊搜尋解析 (需有 Google Key) | `.env` 已設定有效 Google API 金鑰 | 1. 手動輸入模糊字詞「台北101」<br>2. 點擊搜尋 | 不會報錯，系統成功將台北101解析為座標並回傳範圍內的餐廳 |
| **TC-003** | 縣市篩選器測試 | 進入「管理餐廳」介面 | 1. 頂部下拉選單選擇「台北市」<br>2. 改選「台中市」 | 清單即時變化，只顯示該縣市的餐廳 |
| **TC-004** | 管理員新增餐廳與縣市 | 已登入，進入「管理餐廳」介面 | 1. 點擊「新增餐廳」<br>2. 縣市選「台中市」，地址輸入「逢甲夜市」<br>3. 點擊儲存並轉換座標 | 顯示儲存成功，列表中出現帶有 `[台中市]` 標籤的逢甲夜市餐廳 |
| **TC-005** | API Key 降級備援測試 | 將 `.env` 的金鑰改為無效字串 | 1. 在首頁手動輸入完整地址「台北市中正區忠孝西路一段1號」<br>2. 點擊搜尋 | 背景印出 Google 失敗訊息，但前端正常顯示搜尋結果（成功降級 Nominatim） |
